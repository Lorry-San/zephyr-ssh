终端页面重构

---

🚀 Zephyr 终端工作台 - 最终版开发需求文档

核心交互与视觉特效（全局）
*   高斯模糊与隐私保护
    *   拖动时： 当用户长按标题栏拖动窗口时，被拖动的窗口内容即时模糊（高斯模糊），只显示轮廓和标题栏，避免文字重影，提升性能与聚焦感。
    *   最小化/关闭时： 窗口缩小消失的过程中，内容同步渐隐并模糊。
    *   后台运行时： 非当前激活的窗口，内容透明度降低或轻微模糊，突出当前操作窗口。
*   动画曲线： 所有位移动画（栏滑出、窗口移动、缩放、分界线滑动）均采用 Spring/弹性动画，参考 macOS 物理手感。

桌面端 Web 交互逻辑

1. 顶部智能悬浮栏
*   默认状态： 隐藏在屏幕最顶部，仅左右两端各露出一个 2-4px 的“拉手/把手”（视觉上像两个小耳朵或细线），提示用户此处可交互。
*   呼出机制： 点击左侧或右侧的“拉手”，整个顶部栏平滑滑下。
*   收回机制：
    *   手动： 点击栏内的“收回按钮”或再次点击“拉手”。
    *   自动： 检测到用户在终端内开始操作（键盘输入/鼠标点击）后，启动 30秒倒计时，倒计时结束自动滑回顶部隐藏，只留拉手。

2. 栏内内容与排列
*   双方向排列：
    *   左侧区域： 按开启顺序（FIFO）排列。
    *   右侧区域： 按最近使用（LRU）倒序排列。
*   图标样式：
    *   类型色： SSH（绿色）、RDP（蓝色）、VNC（橙色）。
    *   文本： 显示前6位字符 + 省略号。
    *   新建： 列表末尾固定 + 号图标。

3. 窗口视图与布局（动态堆栈逻辑）
系统维护一个内部的窗口开启顺序堆栈，无论用户如何通过拖拽改变视觉位置，窗口的“替换逻辑”始终遵循最初的开启顺序。

*   1个窗口：
    *   居中全屏显示。
*   2个窗口（左右分屏）：
    *   布局： 左右各 50%。
    *   位置逻辑： 最早开启的在左侧，最新开启的在右侧。
    *   分界线： 中间分界线支持左右拖拽，调整左右窗口比例。
*   3个窗口（左大右小）：
    *   布局： 左侧占 2/3，右侧分为上下两个 1/3。
    *   位置逻辑：
        *   左侧 (2/3)： 最新开启的窗口。
        *   右上 (1/3)： 第2个开启的窗口。
        *   右下 (1/3)： 第1个（最早）开启的窗口。
    *   分界线： 支持拖动左侧垂直分界线（改变左右比例），以及右侧水平分界线（改变右上右下比例）。
*   4个及以上窗口（自动最小化机制）：
    *   逻辑： 当开启第 4 个窗口时，触发“堆栈挤出”逻辑。
    *   动作：
        1.  最早开启的窗口（原右下）：自动最小化收回至顶部栏。
        2.  第2个开启的窗口（原右上）：移动至右下角。
        3.  最新开启的窗口（原左侧）：移动至右上角。
        4.  第4个（当前最新）窗口：占据左侧 2/3 主区域。
    *   循环： 此后每新开一个窗口，都重复此“右下最小化 -> 右上移右下 -> 左移右上 -> 新进占左”的逻辑。

4. 拖拽交互与位置记忆
*   窗口拖拽：
    *   长按标题栏的 :: 或空白处可拖动。
    *   拖动到一个已有窗口上方松手，两者交换位置。
    *   拖动到屏幕边缘自动吸附并调整布局。
*   位置记忆与替换（核心规则）：
    *   用户手动拖拽交换窗口位置（例如将左侧大窗拖到右侧）后，视觉布局改变。
    *   但是，当触发“新开窗口导致旧窗口最小化”的逻辑时，系统不依据当前的视觉位置，而是依然依据内部的开启顺序堆栈。
    *   示例： 即使你把最早开启的窗口拖到了左侧，当第4个窗口开启时，被挤出去最小化的依然是那个最早开启的窗口（逻辑上的 Slot 1），而不是当前视觉上的左侧窗口。

5. 窗口标题栏操作（... 菜单）
点击 ... 弹出菜单：
1. 全屏
2. 左半屏
3. 右半屏
4. 右侧 1/3 上半部
5. 右侧 1/3 下半部
6. 左侧 2/3
7. 右侧 2/3
8. 最小化（收回至顶部栏）
9. 关闭

移动端适配

1. 视图限制
*   单窗口模式： 手机/平板端只能同时显示 1 个终端窗口。
*   多开处理： 打开新终端时，当前窗口自动最小化至顶部栏。

2. 顶部栏交互
*   常驻/滑出： 顶部栏逻辑同桌面端（带拉手，可呼出）。
*   菜单简化： 移动端点击窗口 ... 菜单，仅保留三个核心功能：
    1. 全屏（切换横竖屏或纯终端模式）
    2. 最小化
    3. 关闭

工程师技术提示

*   CSS 视觉：
    *   使用 backdrop-filter: blur(10px) 实现全局毛玻璃效果。
    *   拖动时的模糊效果可使用 CSS class 切换配合 transition。
*   布局实现 (CSS Grid)：
    *   建议使用 CSS Grid 实现动态布局。
    *   2窗口： grid-template-columns: 1fr 1fr (通过 JS 动态修改 fr 比例实现分界线拖拽)。
    *   3窗口： grid-template-columns: 2fr 1fr; grid-template-rows: 1fr 1fr;，左侧窗口 grid-row: 1 / 3。
*   状态管理 (State)：
    *   需要维护两个状态数组：visualLayout (当前屏幕显示的 DOM 顺序) 和 openOrderStack (不可见的逻辑开启顺序)。
    *   当触发自动最小化时，读取 openOrderStack[0] 进行移除操作，然后重新计算 Grid 布局。
*   交互库：
    *   窗口拖拽与碰撞检测建议使用 interact.js 或 dnd-kit。
    *   分界线拖拽需监听 onMouseMove 动态计算 Grid 的 fr 值或百分比宽度。
*   倒计时： 顶部栏的“自动收回倒计时”需要在每次全局 onKeyDown 或 onMouseDown 时重置。
