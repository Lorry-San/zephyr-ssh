Bug #12：监控面板底部边角样式不一致（P3 - 低优先级）

问题描述：
“服务器实时监控”浮窗面板底部的左右两个边角目前为直角（或半径过小的圆角），与面板顶部及整体界面的圆角曲率不统一，视觉上显得突兀且割裂。

影响范围：
服务器实时监控面板（及其他同类浮窗组件）的底部边角样式

期望结果：
将面板底部左右两个角的圆角半径（border-radius）调整为与面板顶部及全局容器一致的数值，确保整体轮廓曲率平滑统一，消除直角带来的视觉生硬感。

终端页面重构

---

🚀 Zephyr 终端工作台 - 最终版开发需求文档

核心交互与视觉特效（全局）
- 高斯模糊与隐私保护：
    - 拖动时： 当用户长按标题栏拖动窗口时，被拖动的窗口内容即时模糊（高斯模糊），只显示轮廓和标题栏，避免文字重影，提升性能与聚焦感。
    - 最小化/关闭时： 窗口缩小消失的过程中，内容同步渐隐并模糊。
    - 后台运行时： 非当前激活的窗口，内容透明度降低或轻微模糊，突出当前操作窗口。
- 动画曲线： 所有位移动画（栏滑出、窗口移动、缩放）均采用 Spring/弹性动画，参考 macOS 物理手感。

桌面端 Web 交互逻辑

1. 顶部智能悬浮栏
- 默认状态： 隐藏在屏幕最顶部，仅左右两端各露出一个 2-4px 的“拉手/把手”（视觉上像两个小耳朵或细线），提示用户此处可交互。
- 呼出机制： 点击左侧或右侧的“拉手”，整个顶部栏平滑滑下。
- 收回机制：
    - 手动： 点击栏内的“收回按钮”或再次点击“拉手”。
    - 自动： 检测到用户在终端内开始操作（键盘输入/鼠标点击）后，启动 30秒倒计时，倒计时结束自动滑回顶部隐藏，只留拉手。

2. 栏内内容与排列
- 双方向排列：
    - 左侧区域： 按开启顺序（FIFO）排列。
    - 右侧区域： 按最近使用（LRU）倒序排列。
- 图标样式：
    - 类型色： SSH（绿色）、RDP（蓝色）、VNC（橙色）。
    - 文本：  6字符显示 前6位...。
    - 新建： 列表末尾固定 + 号图标。

3. 窗口视图与布局（最大同屏 3 个）
- 布局规则：
    - 1个： 居中全屏。
    - 2个： 左右分屏（50%/50%）。
    - 3个： 左大右小（左侧占 2/3，右侧上下各占 1/3）。
- 拖拽交互：
    - 长按： 长按标题栏的 :: 或空白处可拖动。
    - 互换： 拖动到一个已有窗口上方松手，两者交换位置。
    - 自动布局： 拖动窗口到屏幕左/右/中边缘，自动吸附并调整布局。

4. 窗口标题栏操作（... 菜单）
- 点击 ... 弹出菜单：
    1. 全屏
    2. 左半屏
    3. 右半屏
    4. 右侧 1/3 上半部
    5. 右侧 1/3 下半部
    6. 左侧 2/3
    7. 右侧 2/3
    8. 最小化（收回至顶部栏）
    9. 关闭

移动端适配

1. 视图限制
- 单窗口模式： 手机/平板端只能同时显示 1 个终端窗口。
- 多开处理： 打开新终端时，当前窗口自动最小化至顶部栏。

2. 顶部栏交互
- 常驻/滑出： 顶部栏逻辑同桌面端（带拉手，可呼出）。
- 菜单简化： 移动端点击窗口 ... 菜单，仅保留三个核心功能：
    1. 全屏（切换横竖屏或纯终端模式）
    2. 最小化
    3. 关闭

工程师技术提示
- CSS: 使用 backdrop-filter: blur(10px) 实现毛玻璃。
- JS: 窗口拖拽建议使用 interact.js 或类似的库处理碰撞检测和吸附。
- Layout: 3窗口布局建议使用 CSS Grid 实现，方便动态切换。
- State: 顶部栏的“自动收回倒计时”需要在每次 onKeyDown 或 onMouseDown 时重置。
